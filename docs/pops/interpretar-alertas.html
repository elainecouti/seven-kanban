Como Ler o Card no Kanban

Exemplo de card:

🔴 Vitoria — 3 problemas

🔴 Conta Suspensa: Verificar cartao/pagamento
🟠 Anuncio Ruim: ads glicada — R$54 gastos, 0 leads
🟢 Escalar Anuncio: ads pancreas — CPL R$7, 8 leads

📅 Verificar: 2026-03-25
📎 Ver relatorio detalhado no link abaixo
Cada card tem:
• Titulo: gravidade mais alta + unidade + numero de problemas
• Bullets: cada problema com emoji de gravidade + resumo curto
• Checklist: separado em ⚡ PRIORIDADE (urgente) e 💡 SUGESTOES (melhoria)
• Link: relatorio HTML detalhado com diagnostico completo, hipoteses e acoes
• Data de verificacao: quando checar se a acao deu resultado

O que Fazer Quando Receber um Alerta

  1. Ler o card completo — entender todos os problemas da unidade antes de agir
  2. Abrir o relatorio detalhado (link no card) — ver diagnostico, hipoteses e dados
  3. Executar as acoes da checklist ⚡ PRIORIDADE — fazer primeiro o que e urgente
  4. Marcar cada acao como feita no checklist do kanban
  5. Depois das prioridades: avaliar as 💡 SUGESTOES e executar o que fizer sentido
  6. Mover o card pra "Fazendo" enquanto trabalha nele
  7. Quando terminar: mover pra "Concluido"
  8. Na data de verificacao: o Intelligence Engine vai comparar metricas antes/depois automaticamente

Tipos de Problema e O que Fazer

Conta Suspensa → Verificar pagamento, trocar cartao, apelar se necessario
Saldo Critico → Avisar cliente pra depositar AGORA
Zero Leads → Checar pixel, LP, publico, criativo
CPL Spike → Analisar por anuncio: qual gastou sem converter?
CPL Alto → Cruzar com funil: se agendamento >20%, manter
CTR Baixo → Criativo nao chama atencao: trocar hook/angulo
Criativo Fadigado → Pausar ad com freq >3.5 + CTR caindo
Anuncio Ruim → Pausar e liberar budget pros bons
Anuncio Top → Escalar: +20% budget ou duplicar ad set
Sem Criativo Novo → Pedir brief pro Design Agent
