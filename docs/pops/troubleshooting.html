🔧 Troubleshooting — Problemas Comuns

Diagnostico rapido e solucao pra cada problema. Consulte antes de entrar em panico.

1. Conta Suspensa / Desativada

Sintomas

Campanha parou de rodar. Status da conta: SUSPENSA ou DESATIVADA no Gerenciador.

  1. Abrir Gerenciador de Negocios → Configuracoes → Contas de anuncio
  2. Verificar se ha notificacao de pagamento pendente (cartao recusado/expirado)
  3. Se for cartao: trocar metodo de pagamento por outro cartao valido
  4. Se for violacao de politica: ler a notificacao, ajustar o criativo/copy e apelar
  5. Apelar em: facebook.com/business/help → "Solicitar analise"
  6. Se demorar >48h: contatar suporte Meta via chat do Business Manager
  7. Plano B: ter conta backup com pixel e audiencias ja configuradas
URGENCIA: cada dia parado = leads perdidos + learning phase resetado. Prioridade maxima.

2. Campanha Reprovada

Sintomas

Anuncio com status DISAPPROVED ou WITH_ISSUES.

  1. Abrir o anuncio reprovado e ler o motivo da reprovacao
  2. Motivos comuns DM2: claims de saude ("cura diabetes"), antes/depois, mencao de condicao medica
  3. Editar o texto: remover claims diretos, usar linguagem indireta ("protocolo focado em...", "busca a remissao...")
  4. Se for imagem/video: verificar se mostra agulhas, sangue, resultados medicos
  5. Salvar e resubmeter — a revisao leva 24-48h
  6. Se reprovar de novo: duplicar o anuncio com copy diferente
  7. Se for falso positivo: apelar via "Solicitar analise"
DM2: evitar palavras "cura", "eliminar diabetes", "parar insulina". Usar "remissao", "controle", "protocolo".

3. Pixel Nao Dispara / Zero Eventos

Sintomas

Events Manager mostra 0 eventos. Campanha nao otimiza. LP nao registra no Supabase.

  1. Abrir a LP no celular (nao no desktop — testar como o usuario real)
  2. Abrir Events Manager → Testar eventos → colar URL da LP
  3. Clicar nos botoes da LP e verificar se PageView + Lead aparecem
  4. Se nao aparecer: verificar se o pixel_id no HTML da LP esta correto
  5. Verificar se fbq('track','Lead') ta no onclick do botao WPP
  6. Verificar se o dominio ta verificado no Business Manager (Configuracoes → Dominios)
  7. Se tudo parecer certo: limpar cache do navegador e testar de novo
Testar com ?exclude=seven desativado (ou em aba anonima) pra nao ser filtrado pelo tracker.

4. Saldo Pre-pago Zerou

Sintomas

Campanhas pausadas automaticamente. Balance = R$0.

  1. Confirmar no Gerenciador: Configuracoes → Pagamento → Saldo
  2. Gerar boleto ou transferir via PIX (opcao mais rapida)
  3. Avisar o cliente imediatamente: "Campanhas pausadas por falta de saldo"
  4. Apos deposito: verificar se as campanhas reativaram automaticamente
  5. Se nao reativaram: ativar manualmente cada campanha
  6. ATENCAO: se ficou parado >3 dias, o learning phase resetou
Impacto: cada dia parado = perda de learning phase + otimizacao do algoritmo. Saldo deve ser monitorado diariamente.

5. Zero Leads com Gasto

Sintomas

Gastou R$50+ em 12h e nao gerou nenhum lead.

  1. Verificar pixel: Events Manager ta recebendo eventos?
  2. Abrir a LP no celular: ta carregando? Botao WPP abre o WhatsApp?
  3. Verificar se a campanha ta otimizando pelo evento certo (Lead, nao PageView)
  4. Checar se o publico nao ta saturado (freq >4)
  5. Verificar se algum criativo foi reprovado
  6. Comparar com dias anteriores: e anomalia ou tendencia?
  7. Se tudo parecer ok: pode ser dia ruim (sabado/domingo tem menos conversao)

6. CPL Explodiu de Repente

Sintomas

CPL de ontem 2x ou mais que a media dos ultimos 7 dias.

  1. Verificar nivel de anuncio: qual ad gastou mais sem converter?
  2. Checar frequencia do ad: se >3.5 com CTR <1% = fadiga. Pausar esse ad.
  3. Verificar se algum criativo top foi pausado acidentalmente
  4. Checar se o CPM subiu (concorrencia no leilao)
  5. Verificar se e sazonalidade (inicio/fim de mes, feriado)
  6. Se for 1 dia isolado: esperar 24h antes de tomar acao drastica
  7. Se for 3+ dias: acao urgente — pausar bottom ads, testar novo angulo
NUNCA pausar tudo por causa de 1 dia ruim. Olhar janela de 3 dias minimo.

7. ZapClinic Login Falhou

Sintomas

Dashboard mostra "Acesso bloqueado" ou "Login falhou" pra uma unidade.

  1. Verificar se a senha mudou (cliente pode ter trocado)
  2. Testar login manualmente no navegador: https://s17.zapclinic.app (ou s18, s22)
  3. Se login funciona no browser mas nao no script: limpar cookies/session
  4. Se senha mudou: pedir nova senha pro cliente e atualizar config.json
  5. Se servidor ta fora (timeout): esperar e tentar mais tarde — s17 e lento
  6. Se problema persistir: contatar suporte ZapClinic
Servidores: s17 (maioria, lento), s18 (Parauapebas, Brasilia), s22 (Anapolis, Colatina).
